## いま止まっているところ（次回の最優先）

### LINE リマインドが `eligible: 0` のまま

**症状（ターミナル）:**
```json
{"sent":0,"failed":0,"eligible":0,"hint":"line_user_id が登録されたユーザーが0人です。SQLで update したか確認してください。"}
```

**想定される原因:**
1. Supabase の `public.users.line_user_id` が、まだ DB に保存されていない（Table Editor で Save し忘れ）
2. SQL を **PowerShell** で実行してしまった（失敗している）— **SQL は Supabase の SQL Editor のみ**
3. `npm run dev` が `.env.local` 更新前のまま動いている（`SUPABASE_SERVICE_ROLE_KEY` 未読込）

**過去の調査結果:**
- 同じ `.env.local` で Node から直接クエリすると、ユーザー 1 件・`line_user_id` あり、と確認済み
- Supabase の SQL Editor で `select` した画面では `U934988bf5...` が入っているスクリーンショットあり
- それでも curl が `eligible: 0` → **開発サーバー再起動**が必要な可能性が高い

### ステップ A: Supabase で DB を確認（ブラウザ・翻訳オフ推奨）

```sql
select id, line_user_id from public.users;
```

- `U934988bf5...` が入っている → ステップ C へ

```sql
update public.users
```

※ ユーザーが複数いる場合は `where id = 'あなたのUUID'` を付ける。

## よくある間違い（再発防止）

| やってはいけないこと | 正しい場所 |
|---------------------|-----------|
| PowerShell で `update` / `select` | Supabase **SQL Editor** |
| `CRON_SECRET` の末尾 `5` を多く付ける（`123455`） | `.env.local` と完全一致（`12345`） |
| `.env.local` 変更後に dev サーバーを再起動しない | 必ず **Ctrl+C → npm run dev** |
| Supabase 画面を Google 翻訳する | 翻訳オフ or シークレットウィンドウ |
